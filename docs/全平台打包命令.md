# 墨形 MorphDraft 全平台打包命令

## 结论

桌面安装包需要在对应操作系统上构建：

- macOS 构建 `.app`、`.dmg`
- Windows 构建 `.exe`、`.msi`
- Linux 构建 `.AppImage`、`.deb`、`.rpm`

macOS 无法通过普通 Tauri 命令可靠生成正式 Windows 和 Linux 安装包。真正一次生成全部桌面平台，需要在 macOS、Windows、Linux 三个构建环境中分别执行命令，通常使用 CI 矩阵。

## 一次性命令

在仓库根目录执行：

```bash
npm run package:all
```

项目打包脚本会自动把 `~/.cargo/bin` 放到 PATH 最前面，确保 Cargo、rustc 和
`rustup target add` 使用同一套 Rust 工具链。不要直接调用 Homebrew 的
`/opt/homebrew/bin/cargo` 构建 Universal macOS 包。

该命令根据当前系统自动执行：

| 当前系统 | 自动构建 |
| --- | --- |
| macOS | macOS Universal；如果移动端已初始化，再构建 Android 和 iOS |
| Windows | Windows；如果 Android 已初始化，再构建 Android |
| Linux | Linux；如果 Android 已初始化，再构建 Android |

移动端尚未初始化时会跳过，不会修改项目。

## 各平台单独命令

### macOS Universal

同时包含 Apple Silicon 和 Intel：

```bash
rustup target add aarch64-apple-darwin x86_64-apple-darwin
npm run package:macos
```

产物：

```text
src-tauri/target/universal-apple-darwin/release/bundle/macos/
src-tauri/target/universal-apple-darwin/release/bundle/dmg/
```

### macOS Apple Silicon

```bash
npm run package:macos:arm64
```

### macOS Intel

```bash
rustup target add x86_64-apple-darwin
npm run package:macos:x64
```

### Windows

在 Windows PowerShell 或 CMD 中执行：

```bash
npm ci
npm run package:windows
```

生成 NSIS `.exe` 和 WiX `.msi`：

```text
src-tauri\target\release\bundle\nsis\
src-tauri\target\release\bundle\msi\
```

### Linux

在 Linux 中安装 WebKitGTK 等 Tauri 系统依赖后执行：

```bash
npm ci
npm run package:linux
```

生成：

```text
src-tauri/target/release/bundle/appimage/
src-tauri/target/release/bundle/deb/
src-tauri/target/release/bundle/rpm/
```

### Android

首次使用先初始化：

```bash
npm run mobile:init:android
```

本机 Android SDK 默认使用：

```text
~/Library/Android/sdk
```

项目脚本会自动设置 `ANDROID_HOME`、`ANDROID_SDK_ROOT`、`NDK_HOME` 和
`ANDROID_NDK_HOME`。当前要求的 NDK 是：

```text
~/Library/Android/sdk/ndk/29.0.13846066
```

打包 APK 和 AAB：

```bash
npm run package:android
```

如需按 ABI 分包：

```bash
npm run tauri android build -- --apk --aab --split-per-abi
```

Android 构建需要配置 Android Studio、SDK、NDK、Java 和签名信息。

### iOS

只能在 macOS + Xcode 环境中构建。

先确认安装的是完整 Xcode，而不是只有 Command Line Tools：

```bash
ls -ld /Applications/Xcode.app
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -license accept
xcodebuild -version
```

安装 CocoaPods：

```bash
brew install cocoapods
```

首次使用先初始化：

```bash
npm run mobile:init:ios
```

生成用于 App Store Connect 的 IPA：

```bash
npm run package:ios
```

本地无签名构建：

```bash
npm run package:ios:unsigned
```

iOS 正式发布需要 Apple Developer Team、证书和 Provisioning Profile。
如果 `security find-identity -v -p codesigning` 显示 `0 valid identities found`，
需要先在 Xcode 的 Settings > Accounts 中登录 Apple Developer 账号并创建证书。

## 只构建当前配置

当前 `src-tauri/tauri.conf.json` 的默认 bundle target 是 `app`。仅按当前配置构建：

```bash
npm run tauri:build
```

这不等同于生成所有安装器。建议正式打包使用上面的平台专用命令。

## 发布前推荐流程

```bash
npm ci
npm test
npm run build
npm run package:all
```

注意：`tauri build` 会再次执行 `npm run build`，因此单独运行前端构建主要用于提前发现问题。
